# ТЗ: Life Planner — Habits + Tasks + Calendar

---

## 1) Идея проекта в одном предложении

**Life Planner** — веб-приложение, где пользователь управляет задачами как в Todoist, отслеживает привычки со streak и прогрессом, и видит всё это в едином календаре.

---

## 2) MVP: что обязательно должно быть

### Модули

* **Tasks** — создавать задачи, расставлять приоритеты, ставить дедлайны, отмечать выполненными, группировать по проектам;
* **Habits** — добавлять привычки, отмечать выполнение, видеть streak и прогресс за неделю/месяц;
* **Calendar** — видеть задачи и привычки на временной сетке по дням/неделям.

### Экраны (навигация через sidebar)

* `/` — Dashboard: сводка на сегодня;
* `/tasks` — список задач в стиле Todoist;
* `/habits` — трекер привычек;
* `/calendar` — календарь с задачами и привычками.

## 4) Детальный функционал по модулям

### Tasks (à la Todoist)

* добавить / редактировать / удалить задачу;
* отметить выполненной (с анимацией и звуком опционально);
* приоритеты: 🔴 срочно, 🟠 высокий, 🟡 средний, ⚪ нет;
* дедлайн с датой и опциональным временем;
* подзадачи (subtasks) с прогресс-баром;
* проекты: создать, переименовать, удалить, цвет/иконка;
* метки (labels): создавать, фильтровать по ним;
* повторяющиеся задачи: ежедневно / еженедельно / ежемесячно;
* виды списка: сгруппированно по проекту, по дате, по приоритету;
* фильтры: сегодня / на этой неделе / все / выполненные;
* поиск по названию задачи;
* перетаскивание задач (drag & drop) для изменения порядка;
* inbox — задачи без проекта.

### Habits

* добавить / редактировать / удалить привычку;
* выбрать цвет и эмодзи-иконку;
* задать цель: X раз в неделю (не только "каждый день");
* отметить выполнение за сегодня / снять отметку;
* streak: считать подряд идущие дни с учётом цели;
* недельный прогресс: X из 7 дней;
* мини-тепловая карта (последние 4 недели);
* мотивационное сообщение при достижении streak (7, 14, 30, 60, 100);
* архив: скрыть привычку, не удаляя историю;
* статистика: общий % выполнения, лучший streak, среднее в неделю.

### Calendar

* три вида: **День / Неделя / Месяц**;
* задачи отображаются на своей дате, кликабельны для редактирования;
* привычки показываются как полоска или кружок под датой;
* drag & drop задачи на другую дату прямо в календаре;
* создание задачи кликом на день;
* сегодня подсвечен, текущий час виден;
* просроченные задачи выделены красным.

### Dashboard (главная)

* привет + дата;
* "Сегодня": список задач на сегодня + привычки на сегодня;
* счётчики: выполнено задач сегодня / активных привычек / лучший streak;
* прогресс-бар дня (% выполненных задач сегодня);
* ближайшие дедлайны (следующие 3 дня);
* быстрое добавление задачи прямо с дашборда.

## 6) Must-have / Must-not

### Must-have

* одностраничное приложение (SPA) без backend;
* localStorage для всех данных;
* тёмная и светлая тема;
* адаптивность: мобайл + десктоп;
* sidebar на десктопе, bottom nav на мобайле;
* плавные анимации переходов;
* пустые состояния для каждого раздела.

### Must-not

* не делать backend и авторизацию;
* не использовать тяжёлые UI-библиотеки (MUI, Ant Design);
* не делать сложную аналитику с графиками BI-уровня;
* не делать мобильное приложение (только адаптивный web);
* не хранить данные в облаке;
* не городить сложный state management (Redux) — хватит zustand или useContext;
* не добавлять AI-фичи.

## 7) UI / Дизайн-система

### Типографика

* **Заголовки**: Fraunces (засечный, с характером);
* **Интерфейс**: Inter или Plus Jakarta Sans.

### Палитра (две темы)

**Светлая**: фон `#FAFAF9`, карточки `#FFFFFF`, акцент `#F59E0B` (янтарь);
**Тёмная**: фон `#1C1917`, карточки `#292524`, акцент `#FBBF24`.

Приоритеты задач: 🔴 `#EF4444` / 🟠 `#F97316` / 🟡 `#EAB308` / ⚪ `#9CA3AF`.

### Карточки и элементы

* скругления 16–20px;
* тени мягкие: `0 2px 12px rgba(0,0,0,0.06)`;
* цветная полоска слева у карточек задач (цвет проекта);
* hover: лёгкий подъём + тень чуть сильнее;
* кнопки "проседают" при нажатии (scale 0.97).

### Анимации

**Использовать**: fade-in, slide-up при появлении, scale при отметке выполнения, smooth height для раскрытия subtasks, blur+fade для модалок.

**Не использовать**: тряску, резкие прыжки, анимации дольше 350ms.

## 8) Edge cases

* задача без названия — запретить сохранение;
* задача без дедлайна — показывать в "Inbox";
* привычка: двойная отметка за один день — игнорировать;
* привычка: streak при цели "3 раза в неделю" — считать по неделям, не по дням;
* некорректные данные в localStorage — сбросить к дефолту с уведомлением;
* очень длинное название задачи/привычки — обрезать с `…`;
* нет задач на день в календаре — показать empty state ячейки;
* смена даты в полночь — пересчитать streak без перезагрузки;
* удаление проекта с задачами — перенести задачи в Inbox;
* повторяющаяся задача — при выполнении создавать следующий экземпляр.

## 11) Definition of Done

* пользователь создаёт задачу, она отображается в списке и на календаре;
* задача с дедлайном сегодня видна на Dashboard;
* привычка отмечается, streak считается по `targetPerWeek`;
* после перезагрузки страницы все данные на месте;
* в тёмной и светлой теме всё читаемо;
* на мобайле не ломается sidebar и карточки;
* пустые состояния есть в каждом разделе;
* удаление проекта переносит задачи в Inbox.
